Big Data Analysis projects using the Python API for Spark(PySpark)

PROJECTS UNDERTAKEN AS PART OF A GRADUATE COURSE AT UCSD

INTRODUCTION

I chose to do a graduate course offered by the Computer Science Department at UCSD: 'Big Data analytics using Spark'. My motivation for undertaking this course as an elective was that the course would give me a good foundation in implementing data analysis and algorithms using a distributed computing framework like Spark. The course has helped me gain hands on experience of writing Python scripts that deploy Spark on an Amazon cluster and perform analytics on data of the order of ~100 GB. The course included 4 projects, each of which introduced application of Spark in a new domain.
I earned an A grade in the course. In the sections below I have included a brief writeup about the problem being analyzed and a link to the code developed to implement the analysis.


  1. Text Processing: Finding the top-5 most frequent n-grams in the text of a novel
  2. Twitter Feed Mining: Top-5 frequently used tokens in the tweets of different groups of users
  3. Implementation of Clustering algorithm: K-means++
  4. Binary Classification using MLlib(Spark's scalable ML library)

Project 1

This was my first PySpark script and hence pretty basic in nature, equivalent to the 'Hello World' script in the usual programming languages. The objective of the script was to parse the novel 'Moby-Dick'(text file) for the various n-grams and count their frequency of occurence. The script outputs a list of the top-5 most frequently occuring unigram, bi-gram, etc., which is a common pre-processing step for text classification. The main takeaway of this exercise was getting familiar with the Map-Reduce paradigm of problem solving, Resilient Distributed Data (RDD) and writing code to deploy Spark framework.

Project 2

This project worked with a set of tweets for different groups of users. The task to accomplish was to list the most popular words (also called tokens) found in the tweets of each group of users. This information would give us a glimpse into what are the common sentiments or opinions of the group of users. I got a chance to work with twitter data in JSON format and manipulate complicated (key,value) pairs during the project.
Code

Project 3

In this project I implemented the K-means++ clustering algorithm by parallelizing independent operations on an Amazon cluster using the Spark framework. The K-means++ algorithm is a variant of the k-means algorithm where the initialization of cluster centroids is done by picking points serially with probability of any point being the next centroid inversely proportional to the square of the distance of the closest centroid already picked.
Code

Project 4

This project involved binary classification on two popular datasets from the 'UCI Machine Learning Repository' namely Covertype and Higgs. MLlib is a library of popular machine learning algorithms implemented on Spark in a scalable manner. I used the Random Forests and Gradient Boosted Trees models from the library to train a classifier on such huge datasets.
